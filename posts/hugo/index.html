一、下载Hugo

  1. 下载Hugo Releases下载对应版本的zip压缩文件。
  2. 下载后解压出来有一个hugo.exe，添加到Path环境变量（也可以直接将hugo.exe放入C:\Windows）。

  3. 打开cmd，运行hugo version，查看 hugo 是否安装成功以及相应的版本号。

    二、开始搭建Hugo博客

  4. 进入Hugo官网，按着Quick Start开始搭建。

  5. 创建一个站点：在cmd中进入一个空目录中，比如C:\Users\tang（cd c:\users\tang），运行下面命令创建站点

    hugo new site myblog

    3.选择一个博客主题，在官网中Themes下面选择一个主题，我的博客用的是m10c，在myblog文件夹中使用git下载主题

    git clone https://github.com/vaga/hugo-theme-m10c.git themes/m10c

    当然也可以不使用git，下载好后将主题放入C:\Users\tang\myblog\themes也行。 然后在config.toml文件（用记事本打开）最后面后添加下面代码：

    theme = "m10c"

    4.创建一篇博客：在cmd进入myblog中执行命令

    hugo new posts/hello-world.md

此时，文章就创建在了 content 文件夹下的 posts 文件夹下了。在编辑好博客后，记得把文章头部的 draft 值改为 false，意思是这已经不是草稿了。 5.启动 Hugo 服务器 在cmd进入myblog目录中,运行下面命令

hugo server -D

复制得到的地址 http://localhost:1313/，在浏览器打开，即可访问到你创建的 Hugo 博客站点，当然，也可以浏览你写的博客这表明你的博客站点在本地上预览是 OK 的 6.构建静态页面：在cmd中执行

hugo

执行完后根目录就会多出一个名为 public 的目录，这就是我们博客站点的静态页面，但是我们不能通过双击的方式预览 public/index.html （ public/index.html 不能使用文件协议预览 ），只能通过

 hugo server -D

预览博客站点。那么我们怎么把它推到 GitHub 上呢？

### 三、部署博客站点静态页面到GitHub

  1. 登录你的github，新建一个Repositories，取名为：yourname.github.io（yourname是你的github名称）

  2. 使用Git Bash将生成的静态页面推到GitHub仓库中，在myblog\public文件夹中使用如下命令：

    git init   #初始化git项目

    然后，

    git add .   
git commit -m '第一次部署'

    接着，

    git remote add origin git@github.com:yourname/yourname.github.io.git
git push -u origin master

在push过程中，需要输入GitHub的账号和密码，此时可以配置SSH（配置方法百度）的方式，省得每次push都要输 这时，本地仓库 public 的内容就推到了远程仓库 http://yourname.github.io，通过http://yourname.github.io便可以访问你的博客了。

四、换头像，添加标签，评论的功能

在每篇文章开头加入如下代码：

+++
title = "Sort1"  # 文章标题
date = 2019-09-20T20:15:35+08:00  # 自动添加日期信息
draft = false  # 设为false可被编译为HTML，true供本地修改
tags = ["算法基础"]  # 文章标签，可设置多个，用逗号隔开。Hugo会自动生成标签的子URL
comments = true  # 是否开启Disqus评论功能
share = true  # 是否开启分享
+++

当然，可以设置每篇文章的默认开头，设置方法如下： 进入C:\Users\tang\myblog\archetypes，修改default.md为：

+++
title = "{{ replace .Name "-" " " | title }}"  # 文章标题
date = {{ .Date }}  # 自动添加日期信息
draft = true  # 设为false可被编译为HTML，true供本地修改
tags = [""]  # 文章标签，可设置多个，用逗号隔开。Hugo会自动生成标签的子URL
comments = true  # 是否开启Disqus评论功能
share = true  # 是否开启分享
+++

同时，还可以修改头像，将你要使用的头像放到C:\Users\tang\myblog\themes\m10c\static，并改名为avatar.jpg，还可以修改C:\Users\tang\myblog下的config.toml，更换简介等，我的配置如下：

baseURL = "https://tangfuping.github.io/"
languageCode = "zh-cn"
title = "汤福平的博客"
theme = "m10c"
paginate = 5

[params]
  author = "汤福平"
  description = "记录算法学习之路"
  [[params.social]]
    name = "github"
    url = "https://github.com/tangfuping"
  [[params.social]]
    name = "twitter"
    url = "https://twitter.com/gohugoio"

在修改完配置后，需要重新生成静态网页和push静态网页，比较麻烦，同时新建文章后也需要重复这个步骤。

五、常用的git操作：

远程协作命令

git checkout dev_xfy git add . git commit -m ‘fskdf’ git push

git checkout dev git pull origin dev_xfy 进去vim i 是进入输入状态 ESC 退到命令状态 在英文输入法下按shift+；底部出现： 输入wq 退出

git安装

git bush here :打开git的命令行工具

git –version:查看git的版本

git的配置

  • git config –global user.name +账号
  • git config –global user.email+邮箱
  • 查看配置信息 git config –list

文件的状态

  • untracked 未被追踪
  • modified 表示工作区修改了某个文件但是还没有添加到暂存区
  • staged 表示把工作区修改的文件添加到了暂存区但是没有提交到版本库
  • committed 表示数据被安全的存储在本地库

git的3层结构

  1. working directory 工作区
  2. staging index 暂存区
  3. git directory（Repository）版本库

git的基本命令

  1. 将文件夹使用git管理 git init
  2. 查看文件夹里的文件 ls -a
  3. 查看文件状态 git status
  4. 将文件从工作区添加到暂存区 git add +文件名
  5. 将文件从暂存区添加到版本库 git commit -m ‘first commit’
  6. 查看提交信息 git log
  7. 将所有文件添加到暂存区 git add .

git的撤销操作

  1. git commit –amend 撤销上一次提交并将暂存区的文件重新提交
  2. git checkout –filename 拉取暂存区的文件并将其替换工作区的文件
  3. git reset HEAD –filename 拉取最近一次提交的版本库中的这个文件到暂存区，改操作不影响工作区

git分支

  1. git branch 查看当前分支
  2. git barach dev 创建dev分支
  3. git checkouk dev 切换分支
  4. git branch -d dev 删除dev分支（必须切换到另一个分支）
  5. git baranch -m dev fix 修改分支名dev为fix
  6. git checkout -b dev 创建分支并切换到该分支

git的指针

  1. HEAD指针总是指向最新版本
  2. HEAD指针指向当前工作的分支
  3. 在切换分支是指向新的分支

git的合并

  1. git merge dev 合并dev分支
  2. 解决冲突：将冲突的内容删除，再使用git commit -am ‘merge dev fixed conflicts’命令
  3. git diff 比较工作区与暂存区文件的差异
  4. git diff –staged 比较暂存区与版本库的文件的差异
  5. git diff 版本号 版本号 比较分支内的两个版本的差异
  6. git diff 分支 分支 比较两个分支的最新提交版本的差异
  7. git stash 暂存工作区的修改到本地
  8. git stash list 查看当前本地保存的文件
  9. git stash apply stash@{0} 将本地保存的文件拉取出来
  10. git stash pop stash@{1} 将本地保存的文件拉取出来且删除此文件
  11. git stash drop stash@{0} 将本地保存的文件删除

git远程仓库

  1. git init 初始化git仓库
  2. git push HTTPS协议 master 将本地文件推送到远程仓库
  3. ssh-keygen 生成一对SSH密钥
  4. 打开生成密钥的文件夹，打开公钥的文件，复制里面的内容
  5. 将复制的内容放在github下SSH keys中
  6. git clone HTTPS 第一次拉取远程仓库的代码
  7. git pull SSH 拉取远程仓库的代码
  8. git remote add github 远程仓库的地址 将远程仓库的地址取别名
  9. git remote -v 查看添加的远程仓库
  10. git push github master 提交修改的代码
  11. git pull github master 拉取远程仓库托管的代码

git文件删除

  1. git rm filename 删除git仓库的文件
  2. git mv oldname newname

码云实战

  1. 首先add,再commit，最后push
  2. git pull origin dev-xfy 切换到其他分支拉取代码
  3. git merge –no-ff dev-xfy 分支合并
